# Repository Architecture

This repository is a small knowledge system.

## Model

- `roadmap.md` defines the learning direction.
- `learning-log/` records history over time.
- Topic folders hold durable notes.
- `AGENTS.md` defines maintenance rules.

## Workflow

1. Search for an existing relevant document.
2. Update that document when possible.
3. Append historical notes to `learning-log/`.
4. Create a new file only for a new topic.
5. Add relative links for easy navigation.

## Principle

Keep the structure simple, explicit, and predictable.
